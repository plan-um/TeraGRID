## 선상 대시보드(BridgeView) 상세 검토

### 설계 철학 재확인

> "위기 상황에서 빠른 상황 인지와 즉각적인 조치를 위해 설계. **분석 도구가 아닌, 운영 인식 및 대응 도구**"

### 🔍 페이지별 상세 분석

#### 8-1-1_선상메인대시보드.html

**✅ 우수한 점**

1. **시각적 임팩트**
   - 위험 상태를 크고 명확하게 표시 (빨간색 배지, 애니메이션)
   - 복원력 상태를 아이콘과 색상으로 직관적 표현
   - 다크 테마로 눈의 피로 최소화

2. **긴급 대응 버튼**
   - 3개의 큰 액션 카드 (사고 대응, 네트워크 격리, 육상 통신)
   - 버튼 크기가 크고 명확함 (터치 가능)

3. **정보 우선순위**
   - 가장 중요한 경보를 상단에 배치
   - 최근 경보 목록을 시간순으로 표시

**⚠️ 개선 필요 사항**

1. **정보 과부하 문제**
   ```
   문제: 한 화면에 너무 많은 정보 (경보 섹션, 액션 그리드, 상태 그리드, 경보 목록)
   영향: 위급 상황에서 집중력 저하
   ```
   
   **개선 제안**:
   ```
   - 위급 상황 모드: 가장 중요한 액션만 표시 (플레이북, 격리 버튼)
   - 정상 상황 모드: 현재와 같은 상세 정보 표시
   - 모드 자동 전환 또는 토글 버튼 제공
   ```

2. **버튼 배치 및 크기 최적화**
   ```
   현재: 3개의 액션 카드가 가로로 배치
   문제: 브릿지 화면에서 좌우 시야각이 넓어 한눈에 보기 어려움
   ```
   
   **개선 제안**:
   ```
   - 화면 중앙에 2x2 또는 1x2 그리드로 재배치
   - 가장 중요한 "플레이북 시작"을 화면 중앙 상단에 배치
   - 버튼 크기를 더 키워서 200px 이상으로 확대
   ```

3. **시각적 계층 구조**
   ```
   문제: 모든 카드가 비슷한 크기와 색상으로 중요도 구분 어려움
   ```
   
   **개선 제안**:
   ```
   - 위급 액션 (플레이북, 격리): 화면의 60% 차지, 큰 버튼
   - 보조 액션 (육상 통신): 화면의 20% 차지
   - 상태 정보: 화면의 20% 차지, 작게 표시
   ```

4. **상황 인식 시간 단축**
   ```
   현재: 복원력 상태를 확인하려면 헤더의 배지와 하단의 상태 카드를 모두 봐야 함
   ```
   
   **개선 제안**:
   ```
   - 헤더에 복원력 상태 아이콘 하나만 크게 표시 (녹색/황색/적색)
   - 클릭 시 상세 정보 팝업으로 표시
   ```

#### 8-1-2_사고경보목록.html

**✅ 우수한 점**

1. **명확한 심각도 표시**
   - 좌측 색상 바와 아이콘으로 시각적 구분
   - Critical, High, Medium 필터링 가능

2. **액션 버튼 직관성**
   - 각 경보마다 명확한 "플레이북 시작" 버튼

**⚠️ 개선 필요 사항**

1. **경보 카드 크기**
   ```
   현재: 카드 높이가 크지만, 중요 정보가 분산
   개선: 제목과 위치만 크게 표시, 상세 정보는 클릭 시 표시
   ```

2. **필터 버튼 위치**
   ```
   현재: 상단 우측에 배치
   개선: 좌측에 배치하여 경보 목록과 가까이 위치
   ```

#### 8-1-3_네트워크상태.html

**✅ 우수한 점**

1. **Zone 상태 명확성**
   - 온라인/격리/오프라인을 색상으로 구분
   - 큰 아이콘과 텍스트로 가독성 우수

2. **긴급 격리 버튼**
   - 하단에 큰 버튼 2개 (전체 격리, 복구)

**⚠️ 개선 필요 사항**

1. **퍼듀 모델 시각화 부족**
   ```
   현재: 텍스트 기반 목록 (Level 5, Level 4, ...)
   문제: 네트워크 계층 구조와 연결 관계가 보이지 않음
   ```
   
   **개선 제안**:
   ```
   - 실제 퍼듀 모델 다이어그램 추가 (피라미드 형태)
   - 각 레벨 간 연결선 표시
   - 트래픽 흐름을 화살표로 표시
   - 차단된 연결은 빨간 X 표시
   ```

2. **Zone별 개별 제어**
   ```
   현재: 전체 네트워크 격리 버튼만 제공
   개선: 각 Zone 카드에 "격리" 버튼 추가
   ```

#### 8-1-4_사고대응가이드.html

**✅ 우수한 점**

1. **체크리스트 기반 플레이북**
   - 단계별 진행 상황 명확
   - 완료된 단계를 시각적으로 구분 (녹색)

2. **진행률 표시**
   - 상단 프로그레스 바로 전체 진행도 파악 가능

3. **단계별 설명 충실**
   - 각 단계마다 수행할 작업과 이유 설명

**⚠️ 개선 필요 사항**

1. **현재 단계 강조 부족**
   ```
   현재: 3번 단계가 파란색으로 표시되지만, 주목도가 낮음
   개선: 현재 단계를 화면 중앙에 크게 표시하고, 
         다른 단계는 접기/펼치기 가능하게
   ```

2. **타이머 및 시간 정보**
   ```
   추가 필요: 각 단계별 예상 소요 시간, 경과 시간 표시
   ```

3. **육상 지원 연락 버튼**
   ```
   추가 필요: 각 단계에서 막힐 경우 즉시 육상 전문가에게 연락할 수 있는 버튼
   ```

#### 8-1-5_원격접속관리.html

**✅ 우수한 점**

1. **활성 세션 명확 표시**
   - 현재 접속 중인 세션을 크게 표시
   - 세션 정보 (IP, 사용자, 경과 시간) 충실

2. **세션 종료 버튼**
   - 즉시 접근 가능한 큰 빨간 버튼

**⚠️ 개선 필요 사항**

1. **테이블 최적화**

   현재: 이력이 테이블 형태로 표시
   문제: 터치 환경에서 테이블은 조작하기 어려움
   개선: 이력을 카드 형태로 변경
   ```

2. **실시간 알림**
   ```
   추가 필요: 새로운 원격 접속 시도 시 팝업 알림 및 승인/거부 버튼

## 육상 대시보드(FleetCommand) 상세 검토

### 📊 개요

육상 대시보드는 **분석 중심의 의사결정 도구**로, 선상 대시보드와는 정반대의 설계 철학을 가져야 합니다:
- **풍부한 데이터**: 다양한 차트, 그래프, 통계
- **심층 분석**: 드릴다운, 필터링, 비교 분석
- **맞춤형 뷰**: 역할별, 관심사별 대시보드 커스터마이징

### 🔧 중기 개선 사항 (1~2주)

1. **선상 대시보드 긴급 모드 구현**
   - 위험 상황 자동 감지 및 UI 전환

2. **퍼듀 모델 시각화**
   - SVG 기반 계층 다이어그램

3. **Chart.js 도입**
   - 경보 추이, 리스크 점수, 선박별 현황 차트

4. **확인 다이얼로그 구현**
   - 위험 액션 시 2단계 확인

5. **실시간 업데이트 표시**
   - 마지막 업데이트 시간 표시
   - 자동 새로고침 옵션

### 🚀 장기 개선 사항 (1~3개월)

1. **실시간 데이터 연동**
   - WebSocket 또는 Server-Sent Events

2. **모바일 앱 버전**
   - React Native 또는 PWA

3. **AI 기반 위협 예측**
   - 머신러닝 모델 연동

4. **다국어 지원**
   - i18n 라이브러리 도입

5. **사용자 행동 분석**
   - Google Analytics 또는 Mixpanel

### 💡 최종 권고사항

#### 우선 순위 1: **메인 대시보드 개선을 가장 먼저** 🚀🚀🚀

**이유**:
```
1. 첫인상이 가장 중요함
   - 사용자가 가장 먼저 보는 페이지
   - 경영진 데모 시 가장 먼저 보여주는 페이지
   - "이 서비스 괜찮네" vs "이거 변변하네"가 여기서 결정

2. 비용 대비 효과 최고
   - 2-3일 투자로 제품 전체의 인상 바꿈
   - Stripe, Linear, Vercel 수준의 보여지는 품질

3. 다른 개선의 기반
   - 지도 API, Chart.js 등을 여기서 먼저 적용
   - 다른 페이지에 재사용 가능
```

**구체적 행동 계획**:
```
Day 1:
✓ KPI 카드 8개로 확장
✓ 실시간 활동 타임라인 추가


Day 2:
✓ Chart.js 도입 및 차트 3개 구현
✓ Leaflet.js 지도 연동

Day 3:
✓ 빠른 작업 섹션 6개 버튼
✓ 최종 테스트 및 디버깅
```

**권장 사항**:
1. 실제 선장 또는 기관장과 함께 시뮬레이션 테스트 진행
2. 랜섬웨어 시나리오를 실제로 진행하며 UI/UX 피드백 수집
3. 야간 브릿지 환경에서 가독성 테스트

**검증 항목**:
- [ ] 5초 안에 위험 상황 인지 가능한가?
- [ ] 10초 안에 격리 버튼 클릭 가능한가?
- [ ] 플레이북을 보면서 실제 조치 가능한가?
- [ ] 터치스크린에서 버튼 오조작 없는가?

#### 우선 순위 2: 백엔드 연동 준비

현재는 정적 HTML이지만, 실제 서비스를 위해서는:

1. **API 설계**
   - RESTful API 또는 GraphQL
   - 엔드포인트 정의 (경보 목록, 선박 상태, 네트워크 제어 등)

2. **데이터베이스 스키마**
   - 선박, 자산, 경보, 취약점, 사용자 등 테이블 설계

3. **인증/인가**
   - JWT 기반 로그인
   - 역할 기반 접근 제어 (RBAC)

#### 우선 순위 3: 성능 최적화

1. **이미지 최적화**
   - WebP 포맷 사용
   - Lazy Loading

2. **CSS/JS 번들링**
   - Webpack 또는 Vite
   - 코드 스플리팅

3. **캐싱 전략**
   - Service Worker (PWA)
   - CDN 활용

5. **데이터 시각화**: 차트 및 그래프 전반적으로 부족
